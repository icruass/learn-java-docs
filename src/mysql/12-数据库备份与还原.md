# 数据库的备份与还原

> **本章导读**
>
> 前面我们花了大量篇幅学习如何往数据库里建表、写数据、设计关系。但有一个问题始终悬在头上：**万一数据没了怎么办？** 误删一张表、硬盘坏了、要把数据从一台机器搬到另一台、上线前想留个还原点……这些都离不开**备份与还原**。
>
> 本章讲清楚：
> 1. 备份的本质是什么（答案：把数据库**导出成一堆 SQL 语句**）；
> 2. 用命令行工具 `mysqldump` 备份、用 `source` / 重定向还原；
> 3. 用图形化工具（SQLyog/Navicat）一键备份还原；
> 4. 备份还原中最容易踩的坑（字符集、还原前要不要先建库）。
>
> 这是一项「平时不起眼、出事救命」的运维技能。本章沿用统一示例库 `db_learn`。

---

## 一、为什么要备份，备份的本质是什么

### 1.1 备份的场景

- **防误操作**：一条没加 `WHERE` 的 `DELETE`/`UPDATE` 就能毁掉整张表（参见第 06 章的警示）。
- **防故障**：硬盘损坏、服务器宕机、机房断电。
- **数据迁移**：把开发库搬到测试库、把本地库搬到服务器。
- **留还原点**：上线前先备份，出问题能秒回滚。

### 1.2 备份的本质

MySQL 最常用的逻辑备份，**本质是把数据库"翻译"成一个 `.sql` 文本文件**，里面是一条条能重新建库建表、插入数据的 SQL：

```sql
-- 一个备份文件 db_learn.sql 内部大致长这样（节选）
DROP TABLE IF EXISTS `dept`;
CREATE TABLE `dept` (
  `id` int NOT NULL AUTO_INCREMENT,
  `dept_name` varchar(20) DEFAULT NULL,
  `loc` varchar(20) DEFAULT NULL,
  PRIMARY KEY (`id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

INSERT INTO `dept` VALUES (1,'研发部','北京'),(2,'市场部','上海'),(3,'财务部','广州');
-- ... emp 表的 DROP/CREATE/INSERT ...
```

所以**还原**就很自然了：把这个文件里的 SQL **重新执行一遍**，数据库就回来了。

> 💡 这种叫**逻辑备份**（导出 SQL）。还有一种**物理备份**（直接复制 data 目录下的数据文件），速度快但不跨版本/跨平台，初学阶段用逻辑备份即可。

## 二、命令行备份：mysqldump

`mysqldump` 是 MySQL 自带的导出工具，**注意它是一个独立命令，要在操作系统的命令行里运行，不是在 mysql 登录后的 `>` 提示符里运行**（这是新手最常犯的错）。

### 2.1 基本语法

```bash
mysqldump -u 用户名 -p 数据库名 > 导出文件路径.sql
```

- `-u`：用户名；`-p`：回车后输入密码；
- `数据库名`：要备份哪个库；
- `>`：操作系统的「输出重定向」，把导出的内容写进文件；
- 文件路径建议写绝对路径，否则文件会落在当前命令行所在目录。

### 2.2 备份单个数据库

```bash
# 在 Windows cmd / PowerShell / Git Bash 里执行（不要先 mysql 登录）
mysqldump -u root -p db_learn > D:/backup/db_learn.sql
```

回车后输入密码，命令执行完没有任何提示就是成功了，去 `D:/backup/` 下能看到 `db_learn.sql`。

### 2.3 其他常用形式

```bash
# 只备份指定的某几张表（库名后跟表名）
mysqldump -u root -p db_learn emp dept > D:/backup/emp_dept.sql

# 备份多个数据库（用 --databases，备份文件里会含 CREATE DATABASE 语句）
mysqldump -u root -p --databases db_learn db_shop > D:/backup/two_db.sql

# 备份所有数据库
mysqldump -u root -p --all-databases > D:/backup/all.sql

# 远程主机备份：-h 指定 IP，-P 指定端口
mysqldump -h 192.168.1.100 -P 3306 -u root -p db_learn > D:/backup/remote.sql
```

> ⚠️ **`--databases` 与不加的区别**：
> - **不加** `--databases`：备份文件里**只有表的 DROP/CREATE/INSERT，没有 CREATE DATABASE**。还原前你必须**先手动建好库并 `USE` 它**。
> - **加** `--databases`：备份文件里**包含 `CREATE DATABASE` 和 `USE`**，还原时会自动建库。

## 三、命令行还原

还原有两种主流方式。

### 3.1 方式一：登录后用 SOURCE 执行

```bash
# 1) 先用 mysql 登录
mysql -u root -p
```

```sql
-- 2) 还原前：若备份文件不含 CREATE DATABASE，需先建库并切换进去
CREATE DATABASE IF NOT EXISTS db_learn CHARACTER SET utf8mb4;
USE db_learn;

-- 3) 用 source 执行备份脚本（路径用正斜杠，或在反斜杠路径外加引号）
SOURCE D:/backup/db_learn.sql;
```

执行后会刷屏一堆 `Query OK`，表示一条条 SQL 在重放。

### 3.2 方式二：不登录，用输入重定向

```bash
# 还原前同样要先有空库（除非备份文件含 CREATE DATABASE）
# 这一步在命令行里完成建库：
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS db_learn CHARACTER SET utf8mb4;"

# 用 < 把 sql 文件喂给指定的库
mysql -u root -p db_learn < D:/backup/db_learn.sql
```

- `<` 是「输入重定向」，把文件内容当作输入交给 mysql 执行；
- 如果备份是用 `--databases` 导出的（含建库语句），可以省略库名：
  ```bash
  mysql -u root -p < D:/backup/two_db.sql
  ```

### 3.3 两种方式对比

| 方式 | 是否需先登录 | 是否需先建库 | 适用 |
|------|--------------|--------------|------|
| `SOURCE 文件;` | 是（在 mysql 内） | 一般要（除非文件含建库语句） | 已经登录、交互式操作 |
| `mysql ... < 文件` | 否（在系统命令行） | 一般要 | 脚本化、自动化还原 |

## 四、图形化工具备份与还原（SQLyog / Navicat）

图形界面的本质和命令行一样（底层就是调用 `mysqldump` 或生成等价 SQL），只是点鼠标更直观。

### 4.1 SQLyog 备份

1. 左侧右键点击要备份的数据库 `db_learn`；
2. 选择 **「备份/导出」→「以 SQL 转储文件备份数据库」**（Backup Database As SQL Dump）；
3. 勾选要导出的内容（结构 Structure / 数据 Data / 两者都要）；
4. 选择导出文件的保存路径，点 **Export**，完成后得到 `.sql` 文件。

### 4.2 SQLyog 还原

1. 先确保目标库存在（没有就新建一个空库）；
2. 右键库 → **「导入」→「执行 SQL 脚本」**（Import → Execute SQL Script）；
3. 选择之前导出的 `.sql` 文件，点执行即可。

Navicat 操作类似：右键库 → 「运行 SQL 文件」还原，或「转储 SQL 文件」备份。

## 五、注意事项与常见坑

- 🕳️ **在哪执行 `mysqldump` 搞错**：`mysqldump` 是**操作系统命令**，必须在 cmd/PowerShell/终端里运行；如果你已经 `mysql -u root -p` 登进去了，在 `mysql>` 提示符里敲 `mysqldump` 会报语法错。反之，`SOURCE` 是 mysql 内部命令，只能登录后用。
- 🕳️ **还原前没建库**：用「不含 CREATE DATABASE」的备份文件还原时，必须**先 `CREATE DATABASE` 再 `USE`**，否则报 `No database selected`。
- ⚠️ **字符集要一致**：备份与还原两端字符集不一致（比如一端 `utf8`、一端 `utf8mb4`）容易导致中文乱码。建库时统一用 `CHARACTER SET utf8mb4` 最稳妥。
- ⚠️ **`mysqldump` 需要在 PATH 里**：如果提示「不是内部或外部命令」，说明 MySQL 的 `bin` 目录没加进系统环境变量 PATH，用绝对路径调用（如 `"C:/Program Files/MySQL/MySQL Server 8.0/bin/mysqldump"`）或把 bin 加到 PATH（参见第 02 章）。
- 💡 **定时备份思路**：把 `mysqldump` 命令写进 `.bat` 脚本，用 Windows「任务计划程序」（或 Linux 的 `crontab`）定时执行，文件名带上日期，实现每日自动备份。

## 六、本章小结

- **备份的本质**：把数据库导出成一个由 `CREATE`/`INSERT` 组成的 `.sql` 脚本（逻辑备份）。
- **备份命令**：`mysqldump -u root -p 库名 > 文件.sql`（在系统命令行执行，不是 mysql 内部）。
- **还原两法**：
  - 登录后 `SOURCE 文件.sql;`
  - 不登录 `mysql -u root -p 库名 < 文件.sql`
  - **两者通常都要先建好空库**（除非备份用了 `--databases`）。
- **图形化工具**：SQLyog/Navicat 右键即可备份/还原，底层等价于上述命令。
- **关键坑**：搞清 `mysqldump`（系统命令）与 `SOURCE`（mysql 内命令）的运行位置、还原前先建库、字符集统一为 `utf8mb4`。
