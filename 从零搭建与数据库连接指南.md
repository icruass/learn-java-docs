# 从零搭建 sms-spring 与数据库连接指南（企业级流程）

> 本文记录从「裸机」到「用 Maven 跑起 Spring Boot 并连上 MySQL」的**完整步骤 + 背后原理**，是我们一步步走过来的复盘。
>
> **本机环境**：Windows 11 · JDK 11 · 后装 Maven 3.9.x + MySQL 8.0 · 客户端用 DBeaver。
> **为什么是 Spring Boot 2.7.x**：Spring Boot 3.x 需要 JDK 17+，而本机是 JDK 11，所以选支持 Java 8/11 的最后一条产品线 2.7.x（用 `javax.*` 命名空间，而非 `jakarta.*`）。

## 1. 先厘清几个概念

| 说法 | 实际是什么 |
|---|---|
| "安装 Maven" | 装一个**构建工具**：负责下载依赖、编译、测试、打包。必须装。 |
| "安装 Spring" | ❌ Spring **不用单独安装**。它只是一堆 jar 依赖，写在 `pom.xml` 里，`mvn` 会自动从中央仓库下载。 |
| "安装数据库" | MySQL 要装（一个独立的服务器软件）；H2 不用装（它是个 jar，随项目走）。 |
| "连接数据库" | 让 Java 程序通过**驱动 + 地址 + 凭证**登录到数据库服务器。 |

一句话：**Maven 和 MySQL 要装；Spring/H2 是依赖，不用装。**

## 2. 工具链安装

### 2.1 确认 JDK（已有 JDK 11）
```bash
java -version          # 看到 11.x 即可
echo %JAVA_HOME%       # CMD；Git Bash 用 echo $JAVA_HOME
```
> ⚠️ **常见坑**：`JAVA_HOME` 必须指向 JDK **根目录**，例如
> `C:\Program Files\Java\jdk-11.0.16` —— **结尾不能带 `\bin`**，否则 Maven 报
> `JAVA_HOME is set to an invalid directory`。

### 2.2 安装 Maven
1. 到 https://maven.apache.org/download.cgi 下载 **Binary zip**（`apache-maven-3.9.x-bin.zip`）。
2. 解压到不带中文/空格的路径，例如 `D:\maven`。
3. 配环境变量（Win11 搜「编辑系统环境变量」→ 环境变量）：
   - 新建 `MAVEN_HOME` = `D:\maven`
   - 编辑 `Path`，新增 `%MAVEN_HOME%\bin`
4. **开新终端**验证（旧终端不会刷新环境变量）：
   ```bash
   mvn -v
   ```
   应输出 Maven 版本、Maven home、以及正确的 `Java version` 和 `JAVA_HOME`。

> **Maven 是干嘛的**：管依赖（自动下 jar）、按标准目录结构编译、跑测试、打包成 jar。企业里几乎不靠「在 IDE 点运行」，而靠 Maven 的生命周期命令。

### 2.3 安装 MySQL 8
1. 到 https://dev.mysql.com/downloads/installer/ 下载体积大的 **MySQL Installer**（完整离线版），底部「No thanks, just start my download」直接下。
2. 安装类型选 **Developer Default**（含 MySQL Server + Workbench 图形客户端）。
3. 配置 Server 的关键项：
   - **Type and Networking**：Development Computer，端口 **3306**。
   - **Authentication**：默认「Use Strong Password Encryption」（MySQL 8 默认，驱动已兼容）。
   - **Accounts**：设置 **root 密码**（本示例用 `root`）。
   - **Windows Service**：勾选配置为服务 + 开机自启（服务名默认 `MySQL80`）。
4. 验证：开始菜单「MySQL 8.0 Command Line Client」输密码进入，敲 `SHOW DATABASES;`；或用 **DBeaver** 连 `localhost:3306` / `root` / `root`。

## 3. 用 Maven 构建与运行（生命周期）

Maven 的核心是一条**生命周期**主线，**执行某阶段会自动先跑它前面的所有阶段**：
```
validate → compile → test → package → verify → install → deploy
```

常用命令（在项目根目录 `sms-spring/` 下执行）：
```bash
mvn clean            # 删除 target/，从干净状态开始
mvn test             # 编译 + 跑测试
mvn package          # 打成可执行 jar（target/sms-spring-0.0.1-SNAPSHOT.jar）
mvn spring-boot:run  # 开发期直接启动服务
java -jar target/sms-spring-0.0.1-SNAPSHOT.jar   # 跑打好的 jar（更接近线上）
```

### `pom.xml` 的三块关键内容
- **`<parent>` spring-boot-starter-parent**：统一管理依赖版本和插件，自己写依赖时大多不用写版本号。
- **`<dependencies>` 各种 starter**：`web`（REST）、`data-jpa`（ORM）、`validation`（校验）、`h2`/`mysql-connector-java`（驱动）。
- **`spring-boot-maven-plugin`**：让 `mvn spring-boot:run` 可用、并把项目打成可独立运行的 jar。

> ⚠️ **踩过的坑**：Spring Boot 的 BOM **并不管理所有依赖**。`mysql:mysql-connector-java` 就没被管理，必须**显式写版本**（本项目用 `8.0.33`），否则报
> `'dependencies.dependency.version' for mysql:mysql-connector-java:jar is missing`。

> ⚠️ **编码坑**：Windows 默认字符集是 GBK。`pom.xml` 里要有
> `<project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>`，否则中文源码/字符串可能乱码。

## 4. 数据库连接原理（核心）

### 4.1 一个 Java 程序连数据库，需要这 4（+1）样东西

| # | 东西 | 说明 | 在本项目哪里 |
|---|---|---|---|
| ① | **驱动 Driver** | Java 用 **JDBC** 作统一接口标准，但各数据库协议不同，要各自的驱动 jar。连 MySQL 用 `mysql-connector-java`（提供 `com.mysql.cj.jdbc.Driver`） | `pom.xml` 依赖 |
| ② | **连接地址 JDBC URL** | 告诉程序「库在哪、怎么连」 | `application.yml` 的 `spring.datasource.url` |
| ③ | **凭证** | 用户名 / 密码 | `username` / `password` |
| ④ | **连接池** | 预开一批连接反复复用，而非每次查询新开。Spring Boot 默认用 **HikariCP** | 自动 |
| +⑤ | **方言 Dialect** | 告诉 Hibernate 按哪种数据库的语法生成 SQL | `database-platform: ...MySQL8Dialect` |

### 4.2 JDBC URL 拆解
```
jdbc:mysql://localhost:3306/sms?useSSL=false&serverTimezone=Asia/Shanghai&characterEncoding=utf8&allowPublicKeyRetrieval=true&createDatabaseIfNotExist=true
```
- `jdbc:mysql://` —— 用 JDBC、走 mysql 驱动
- `localhost:3306` —— 数据库在本机、3306 端口
- `/sms` —— 连名为 `sms` 的库
- 参数：
  - `createDatabaseIfNotExist=true` —— 库不存在就自动建（所以没手动建 `sms` 也能用）
  - `serverTimezone=Asia/Shanghai` —— 时区（不写有的版本报错）
  - `characterEncoding=utf8` —— 按 UTF-8 传输，中文不乱码
  - `useSSL=false` / `allowPublicKeyRetrieval=true` —— 本地关 SSL、配合 MySQL8 密码加密方式取公钥

### 4.3 Spring Boot 怎么「自动连上」
传统 JDBC 要手写：
```java
Class.forName("com.mysql.cj.jdbc.Driver");
Connection c = DriverManager.getConnection(url, user, password);
```
Spring Boot 帮你省了：启动时它发现「classpath 有 JDBC 驱动 + 你配了 `spring.datasource.*`」，就**自动**创建一个带 HikariCP 连接池的 `DataSource` 并注入。你一行连接代码都不用写——这叫**自动配置（auto-configuration）**。

### 4.4 profile：一套代码切不同数据库
- `src/main/resources/application.yml` 是默认配置。
- 可以再放 `application-{名字}.yml` 作为某个 profile 的配置。
- 启动加 `--spring.profiles.active=名字`，该 profile 的配置会**覆盖**默认值。
- 这就是「换数据库不改 Java 代码、只切配置」的原理。
  > 本项目最终把默认配置直接设成了 MySQL（见第 5 节），所以**日常启动不需要任何 profile 参数**。

## 5. 从 H2 切换到 MySQL（我们走过的路）

### 5.1 为什么一开始用 H2
H2 是**用 Java 写的内存数据库**，零安装、随应用启停、讲 SQL、走 JDBC。适合**开发演示和跑测试**：启动快、自包含。代价是数据在内存里、**重启即丢**，所以生产不用它。

### 5.2 H2 当初「长」在 5 个地方
1. `pom.xml` 的 `com.h2database:h2` 依赖
2. `application.yml` 的 H2 数据源 + `h2.console` + `sql.init`
3. `data.sql`（H2 启动时自动灌的种子数据）
4. `application-mysql.yml`（为「切换」而存在的第二份配置）
5. 两个 `@SpringBootTest` 测试类（默认读 `application.yml`，即跑在 H2 上）

### 5.3 改造目标：应用只用 MySQL、H2 仅留给测试（企业标准做法）
> 原则：**应用用 MySQL（数据要持久）；测试用 H2 内存库（要快、要自包含、不依赖本机 MySQL）**。

**步骤**：
1. **改默认配置为 MySQL** —— 把 `src/main/resources/application.yml` 改成 MySQL 数据源（url/driver/username/password + `ddl-auto: update` + `MySQL8Dialect`）。
2. **删切换用配置** —— 删除 `application-mysql.yml`（默认已是 MySQL，不再需要 `--spring.profiles.active`）。
3. **删 H2 自动种子** —— 删除 `src/main/resources/data.sql`（MySQL 的种子用 `db/mysql-seed.sql` 手动灌一次即可）。
4. **H2 降级为仅测试** —— `pom.xml` 里给 H2 依赖加 `<scope>test</scope>`：打包/运行应用时不含 H2，只有 `mvn test` 时才有。
5. **给测试单配 H2** —— 新建：
   - `src/test/resources/application.yml`：指向 H2 内存库（`create-drop` + `defer-datasource-initialization: true` + `sql.init.mode: always` + `encoding: UTF-8`）。
   - `src/test/resources/data.sql`：测试用种子数据（2 教室 + 2 老师）。
   > 跑测试时 `src/test/resources` 的配置会**盖过** `src/main`，于是测试自动用 H2，和你的 MySQL 互不干扰。

> ⚠️ **踩过的坑**
> - `spring.sql.init.mode: always` **不允许 `data.sql` 是空文件**，否则报 `'script' must not be null or empty`。要么文件有内容，要么干脆别建。
> - 读 `data.sql` 默认用平台字符集（Windows=GBK）→ 中文乱码。要加 `spring.sql.init.encoding: UTF-8`。
> - 实体（`@Entity`）的 setter **不能是 `final`**，否则 Hibernate 代理报 `Setter methods of lazy classes cannot be final`。

## 6. 验证与排错

### 6.1 确认应用真的连上了 MySQL
看启动日志：
- ✅ `HikariPool-1 - Start completed`
- ✅ URL 是 `jdbc:mysql://localhost:3306/sms`
- ✅ `Started SmsApplication in X seconds`

### 6.2 持久化验证（MySQL vs H2 的本质区别）
注册一个学生 → 在 DBeaver 刷新 `sms.student` 看到它 → **停掉应用、重启、再查，数据还在** = 真正落盘（H2 重启就没了）。

### 6.3 连接常见错误对照
| 报错 | 多半原因 |
|---|---|
| `Access denied for user 'root'` | 用户名/密码错 |
| `Communications link failure` | MySQL 没启动 / 主机或端口错 |
| `Unknown database 'sms'` | 库名错（本项目有 `createDatabaseIfNotExist`，一般不会） |

### 6.4 看测试报错的正确姿势
失败后看 `target/surefire-reports/*.txt`：
- **ERROR** = 上下文没起来（如配置/连接问题）；**Failure** = 跑起来了但断言不过。
- 重点看 **`Caused by`** 那几行，它指向真正的根因。

## 7. 常用命令速查

```bash
# 验证工具链
java -version
mvn -v

# 构建 / 测试 / 运行（在 sms-spring/ 下）
mvn clean test                                   # 跑测试（用 H2）
mvn clean package                                # 打 jar
mvn spring-boot:run                              # 启动（默认连 MySQL，无需 profile）
java -jar target/sms-spring-0.0.1-SNAPSHOT.jar   # 跑 jar

# 手动给 MySQL 灌种子（首次需要时；UTF-8 避免中文乱码）
mysql -uroot -proot --default-character-set=utf8mb4 sms < src/main/resources/db/mysql-seed.sql

# 调接口
curl http://localhost:8080/api/classrooms
curl -X POST http://localhost:8080/api/students -H "Content-Type: application/json" -d "{\"name\":\"Tom\",\"gender\":\"MALE\",\"age\":8,\"classroomId\":1}"
```
> Windows 命令行里 `curl -d` 直接写**中文**会乱码报 400（跟应用无关）；测中文用 Postman、或把 JSON 存成 UTF-8 文件再 `curl --data-binary @文件`，或用浏览器同源 `fetch`。
